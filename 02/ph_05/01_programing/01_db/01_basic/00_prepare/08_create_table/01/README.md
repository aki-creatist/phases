# テーブルの意味

* DBのテーブルを考える時、Exelのシートのような１枚の表をイメージする
    * 縦横に罫線が入って、項目名が一番上にあるような表
* 保存データは項目ごとに、表の「列」に当たるカラムに格納される
    * カラム: またはフィールと呼ぶ
* カラムごとに名前があり、データの型や長さを指定可能
* 横に並んだ各カラム(表の「行」に当たる部分)が集まって、１件のデータまたは１レコードになる
* このようなDBの構造を`スキーマ`と呼ぶ
